# CONTRIBUTING

Thank you for your interest in contributing to this project.

This repository is part of a DSCI 310 course project. We welcome bug reports, suggestions, documentation improvements, and code contributions that improve the project’s clarity, reproducibility, and analysis.

## How to Contribute

### Reporting bugs or suggesting improvements

If you notice a bug, unclear documentation, or have an idea for improvement, please open a GitHub issue. When creating an issue, please include:

* a clear and descriptive title
* a short explanation of the problem or suggestion
* steps to reproduce the issue, if applicable
* any relevant screenshots, outputs, or context

## Making contributions with pull requests

We use GitHub Flow for this project.

To contribute code or documentation:

1. Create a new branch from `main`
2. Make your changes
3. Commit with a meaningful commit message
4. Open a pull request into `main`
5. Request review from at least one teammate before merging

Please keep branches focused on one main task whenever possible.

### Branch naming

Use clear and descriptive branch names such as:

* `add-code-of-conduct`
* `write-readme`
* `setup-dockerfile`
* `analysis-eda`
* `model-knn`

## How We Organize Work

We organize project tasks using GitHub issues, milestone planning, and pull requests.

To keep the repository structured and easy to review:

* work is divided into small, independent branches
* issues and pull requests are used to categorize tasks such as documentation, setup, analysis, and modeling
* changes should be documented clearly so other team members can understand and reproduce them

## Collaboration Style

This is a team project, and some parts of the coding and planning may be done collaboratively in a group setting.

When working together, we still aim to:

* keep commits meaningful
* describe decisions clearly in issues and pull requests
* make sure all teammates can understand, review, and run the work

## Code of Conduct

All contributors must follow the project [Code of Conduct](CODE_OF_CONDUCT.md).

## Questions

If you are unsure where to start, please open an issue and we will do our best to respond.
